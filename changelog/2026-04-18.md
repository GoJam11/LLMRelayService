# 2026-04-18

### 新增

- **Model Routes（模型别名路由）**：新增 `model_aliases` 数据库表，支持将虚拟模型名（如 `Auto`）映射到任意 Provider + 真实模型。客户端发送 `model: "Auto"` 时，网关优先查别名表，匹配后改写请求体中的 model 字段并转发至目标 Provider；若别名 Provider 不可用则自动降级为普通模型查找。前端新增独立 **Routes** 页面（导航：Monitor / Usage / Providers / Models / Routes / Keys / Logs），支持新建、编辑、启用/禁用、删除别名，并对 Provider 已删除或模型已移除的失效别名展示警告标记。后端新增 `/__console/api/model-aliases` CRUD API。

- **Provider 编辑弹窗左右布局**：将原来的单列滚动表单拆分为左右两栏：左侧为 Channel 基础配置（名称、类型、URL、Priority、System Prompt、Auth），右侧独立展示并管理模型列表（含 Sync from Upstream），更直观。

### 变更

- **显式路由前缀改为 `/providers/channelName`**：原 `/{channelName}/v1/...` 显式前缀路由改为 `/providers/{channelName}/v1/...`，避免与 `/openai`、`/anthropic` 等类型强制前缀及其他一级路径冲突。Provider 编辑弹窗中的路径提示文字同步更新。

- **models.dev 数据持久化到 DB + 并行启动预热**：新增 `model_catalog_cache` 数据库表，统一缓存 context window 和 pricing 数据。`model-catalog.ts` 与 `pricing.ts` 共享一个 `catalog-db.ts` 层和同一个网络请求（`fetchModelsDevData`），避免双倍拉取。启动时先从 DB 加载缓存到内存（毫秒级，不阻塞），若 DB 缓存已过期则在服务启动后异步后台刷新；`ModelPricing` 类型迁移至 `catalog-db.ts` 以消除循环依赖。

- **Routes 页面每行新增测试按钮**：点击烧瓶图标可测试该别名对应 Provider + 模型的连通性，复用 Provider 测试接口。测试结果以绿色 ✓ / 红色 ✗ 图标显示，hover 展示延迟和错误信息。
