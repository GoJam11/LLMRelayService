# 2026-06-24

### 新增
- 渠道编辑弹窗的模型列表区域，在 Sync / Add 同级新增「清空」按钮（destructive ghost 样式 + Trash2 图标），点击后二次确认即可一次性清空当前渠道的全部模型，便于在 Sync 拉取上游模型前先重置本地列表；按钮在当前模型列表为空时自动禁用。新增对应中英文 i18n 文案 `providers.clearAllModels` / `providers.clearAllModelsConfirm`。

### 变更
- 流式请求上游首字节超时的默认值从 `30s` 提升到 `300s`，与普通 / 图片请求保持一致；同步更新 `CODE_DEFAULT_GATEWAY_TIMEOUTS.streamFirstByteTimeoutMs`、控制台「网关超时配置」页表单默认值、README 文档说明以及对应单元测试（用显式覆盖 `streamFirstByteTimeoutMs` 的方式验证流式分支选择逻辑，避免和默认值撞车）。
- 上游首字节 504 超时响应的 `detail` 文案在单条路由发生多次重试时，会附带尝试次数和累计时长（如 `No first byte received within 300s per attempt (3 attempts, ~900s total)`），单次尝试场景保持原文案 `No first byte received within 300s`，便于日志侧快速识别"是单次慢"还是"多次重试都慢"。
