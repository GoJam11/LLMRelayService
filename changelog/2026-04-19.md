# 2026-04-19

### 新增
- 为 `console_providers` 表添加 `provider_uuid` 字段（stable UUID），通过内联迁移 0014 自动生成，用于稳定追踪渠道身份
- 模型别名（model aliases）的 `provider` 字段改为存储渠道 UUID，不再依赖 `channelName`（可变字符串），防止渠道重命名导致别名失效
- 路由解析层新增 UUID→channelName 映射表（`uuidToChannelName`），支持 UUID 或 channelName 双模式查找（兼容旧数据）
- 后端 `ProviderInfo` 接口新增 `providerUuid` 字段，前端 `ProviderInfo` 类型同步更新
- 历史无 UUID 的渠道行在首次 `listConsoleProviderEntries()` 时自动回填随机 UUID（backfill）
- 前端路由页（routes-page）创建/编辑别名时，provider 下拉存储 `providerUuid`；表格展示时反查 channelName；测试按钮调用时自动解析 UUID→channelName
- **前端全量中英文 i18n**：基于 react-i18next + i18next，覆盖所有 16 个页面/组件文件（nav-bar、session-shell、logs-page、dashboard-page、usage-page、request-log-table、detail-view、detail-page、providers-page、routes-page、models-page、keys-page、bucket-table、channel-health、payload-panel、usage-trend-chart），约 400+ 文案字符串。语言切换按钮位于导航栏，默认跟随浏览器语言、回退中文，选择保存在 localStorage。

### 变更
- 日志页面移除 `message_roles` 参数传递（后端已标记为可选，前端不再使用）
- 日志页面移除自动刷新功能，避免与手动翻页冲突
- **统一所有页面标题布局**：将页面标题从 Card 内部移到独立 flex 行，每个页面配有专属 Lucide 图标（Dashboard=Activity, Usage=BarChart3, Logs=ScrollText, Providers=Server, Routes=GitFork, Keys=KeyRound, Models=BookOpen），移除 Badge 标签和大号标题，统一为 `text-base font-semibold` + `text-xs text-muted-foreground` 描述。筛选面板独立放入单独 Card。
- 更新 design guide（SKILL.md）添加 Page Header 和 Filter Panel 标准布局规范
- **提取 `<PageHeader>` 组件**（`@/components/ui/page-header`）：将所有页面内联的标题布局统一为可复用组件，接受 `icon`/`title`/`description`/`actions` props，7 个页面全部切换使用
- Keys 页面移除多余的 Card 包裹，表格直接渲染

### 新增（续）
- 日志页面新增 **Live 模式**：点击右上角 Live 按钮后，清空当前日志并每 3 秒轮询最新 100 条记录，不显示分页；再次点击退出 Live 模式

### 修复
- Monitor 页面筛选区移除多余的内层 `rounded-xl border bg-secondary/10` 容器，与 Usage/Logs 页面统一为单层 Card 结构
- 更新 design guide 明确禁止 filter Card 内再嵌套样式容器

### 变更（续）
