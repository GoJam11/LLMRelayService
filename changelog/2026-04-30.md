# 2026-04-30

### 新增

- **API Key 模型限制**：新增通过 API key 限制可使用模型的白名单机制，前后端全实现。
  - 数据库 `console_api_keys` 表新增 `allowed_models_json` 列（JSON 字符串数组，空数组=不限制），通过启动时 inline migration 自动迁移（迁移编号 0015）。
  - 后端 `src/api-keys.ts`：`StoredApiKeyRecord` 和 `AuthenticatedApiKeyInfo` 均增加 `allowed_models: string[]` 字段；新增 `setApiKeyAllowedModels(id, models)` 函数；`authenticateManagedApiKey` 返回携带 `allowed_models` 的完整信息。
  - 后端 `src/index.ts`：认证成功后，若 API key 配置了非空 `allowed_models`，则对请求的模型名执行白名单校验（支持 `*` 尾部通配符，如 `claude-*`），不在白名单内返回 403。
  - 后端 `src/console-ui.ts`：新增 `PATCH /__console/api/keys/:id/allowed-models` 接口，接受 `{ models: string[] }` 更新白名单。
  - 前端 `types.ts`：`ManagedApiKey` 新增 `allowed_models: string[]` 字段。
  - 前端 `api.ts`：新增 `setKeyAllowedModels(id, models)` 接口调用函数。
  - 前端 `keys-page.tsx`：在密钥列表中新增"模型限制"列（显示不限制/限N个），每行增加"模型限制"操作按钮，点击弹出 Dialog 可添加/删除允许模型，支持回车输入、badge 标签展示与逐项删除。
  - i18n（zh/en）：新增模型限制相关翻译字符串。
