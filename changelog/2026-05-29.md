# 2026-05-29

### 新增

- **端到端路由 & 故障转移测试**（`test/gateway-routing-failover-e2e.test.ts`）：
  - 使用 `Bun.serve({ port: 0 })` 启动本地 mock upstream server，完全不依赖真实 AI 服务
  - 覆盖 28 个场景，分 6 个类别：
    1. **路由**：模型匹配、显式 /providers/:name/ 路径、未知模型 400、优先级选择、disabled provider 跳过、类型前缀限制
    2. **认证**：缺少 key → 401、错误 key → 401、正确 key → 正常代理
    3. **故障转移—状态码重试**：5xx 重试后成功、全部重试耗尽、429 重试、400/403 不重试、streaming 请求不重试状态码
    4. **故障转移—网络/超时**：网络错误回退到备用 provider、无回退返回 502、超时重试后成功、全部超时 → 504、retryOnTimeout=false 不重试
    5. **模型回退**：same_model 模式、any_model 模式（body 中 model 字段被重写）、customModelFallbacks 自定义规则、全部失败透传最后 upstream 错误、maxFallbackAttempts 限制回退次数
    6. **策略控制**：failover disabled 立即透传、maxFallbackAttempts=0 禁用回退但允许同 provider 重试

- **测试注入接口**（`src/gateway-timeouts.ts`）：新增 `forceTimeoutSettingsForTest`，测试环境下直接注入 timeout 设置并绕过最小值校验，用于在测试中模拟极短超时（如 200ms）触发超时路径

- **测试注入接口**（`src/gateway-failover.ts`）：新增 `loadFailoverPolicyForTest`，测试环境下直接注入故障转移策略并绕过数据库

### 修复

- **`src/api-keys.ts`**：`authenticateManagedApiKey` 增加 try-catch 包裹 DB 查询，数据库不可用时返回 null（而非抛出异常导致 500），修复了测试环境和生产环境中 DB 暂时不可用时认证失败会返回 500 而非 401 的 bug；同时修复了原代码 `return false` 的类型错误（应为 `return null`）
