# 2026-07-02

### 新增

- 支持 SQLite 部署（与 PostgreSQL 二选一，首次部署由 `DATABASE_URL` 选定方言，选定后不可切换）：
  - 不设置 `DATABASE_URL` 时默认使用 `./data/llm-relay.sqlite`（bun:sqlite，WAL 模式），单容器零依赖即可部署，无需 compose 和外部数据库；`sqlite:<路径>` 可自定义数据文件位置
  - schema 拆分为 `src/db/schema.pg.ts` / `src/db/schema.sqlite.ts` 双方言定义，`src/db/schema.ts` 运行时按方言导出表对象，上层查询代码零改动；新增 `drizzle.sqlite.config.ts` 与 `drizzle-sqlite/` 迁移目录（`db:generate:sqlite` 等脚本）
  - `runMigrations()` 按方言选择迁移器（PG 走 advisory lock + `drizzle/`，SQLite 走 `drizzle-sqlite/`）；控制台数据库重置 API 同步支持 SQLite（drop 全表后重新迁移）
  - API Key 配额扣费的 PG 专属 CTE（`FOR UPDATE` + `UPDATE...FROM`）增加 SQLite 分支，用 `BEGIN IMMEDIATE` 事务实现同等原子性
  - 新增方言守卫 `src/db/dialect-guard.ts`：首次启动把方言写入数据目录 `.db-dialect` 标记，之后启动校验，误切换方言时拒绝迁移并在降级页给出恢复指引
  - Dockerfile 预创建 `/app/data` 并打包 SQLite 迁移文件，挂载 `-v <volume>:/app/data` 即可持久化；README（中英）新增「单容器 SQLite」部署指引，`.env.example`、`docker-compose.yml`、`src/db/README.md`、`AGENTS.md` 同步更新双方言说明
