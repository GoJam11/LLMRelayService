# 2026-06-02

### 新增
- 请求日志和详情页新增 retry 标注：当请求触发同路由重试时，DB 新增 `retry_attempt` 整型列（默认 0）记录重试次数（0 = 首次成功，1 = 经历 1 次重试后成功，以此类推）。
- 终端 `[REQ]` 日志在 `retry_attempt > 0` 时自动附加 `retry_attempt` 字段，方便在容器日志中直接识别重试请求。
- Dashboard 日志列表对触发了同路由重试的请求展示「已重试」badge（与跨路由回退的「已回退」badge 区分）。
- Dashboard 请求详情页新增「重试次数」行，仅在 `retry_attempt > 0` 时显示。
- 新增 drizzle 迁移 `drizzle/0006_equal_power_man.sql`，为 `console_requests` 表添加 `retry_attempt INTEGER NOT NULL DEFAULT 0` 列。
