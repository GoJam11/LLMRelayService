# 2026-04-26

### 新增
- 渠道配置新增 OpenAI Responses 处理方式，支持原生转发、转换为 Chat Completions、禁用三种模式；控制台表单和渠道卡片同步展示该策略。
- 日志表请求列为 Responses 转 Chat Completions 的兼容请求新增“已转 Chat”标签，并用 tooltip 展示转换说明与实际上游目标地址。

### 变更
- OpenAI `/v1/responses` 默认改为原生转发，只有渠道显式选择兼容转换时才会改写到上游 `/chat/completions`。

### 修复
- 当上游响应没有返回 token usage 时，响应记录阶段改用转发请求体和响应文本估算 input/output tokens，并写入 `estimated` 标记，避免用量与成本统计记录为 0。
- 优化 token 估算器，从 OpenAI/Anthropic JSON 与 SSE 响应中提取真实 assistant 文本再计数，避免把完整 JSON/SSE envelope 当作输出内容计费。
- 修正转发请求体作用域，确保响应日志异步保存时仍能拿到最终发送给上游的请求体用于输入 token 估算。
- 移除请求详情页未使用的 `sourceRequestType` 局部变量，修复部署构建时 `tsc -b` 因 `noUnusedLocals` 报错失败的问题。
- 新增仓库级 pre-commit hook，提交前运行 `bun run build`，并提供 `bun run hooks:install` 用于配置本地 `core.hooksPath`。
- 将 `console_requests.token_usage_estimated` 字段和索引加入启动 inline migrations，服务启动时自动补齐缺失 schema，避免请求日志写入因未手动迁移而报错。
- 新增独立 OpenAI Responses/Chat Completions 兼容模块，将 `/v1/responses` 请求转换为上游 `/chat/completions` 请求，并把非流式与 SSE Chat 响应包装回 Responses 结构。
- 日志表新增单请求缓存命中率列，并在 token 用量为兜底估算时于 Token 数列追加“估算”标记。
- 修复 Responses 到 Chat Completions 兼容转换在本地返回 400 时未写入日志的问题，现在会记录原始请求、错误响应体与兼容错误原因。
- Responses 到 Chat Completions 兼容转换会忽略 `web_search` 等 Chat Completions 无法表达的内置工具，避免因 built-in tool 直接返回本地 400。
